LifeCity 2.5D — Modular Refactor V1

Цель:
Разнести клиентский код по понятным модулям без изменения серверной логики и без поломки текущего запуска.

Что изменено:
1. jobs.js больше не хранит всё подряд.
   Теперь:
   - public/js/jobs-system.js — логика работ, действий, коллизий с зданиями.
   - public/js/city-layout.js — данные города: дороги, районы, зоны, парки, тротуары, переходы, городской декор.
   - public/js/roads-render.js — отрисовка дорог, разметки, переходов, парков и зон.
   - public/js/jobs.js — оставлен как legacy-placeholder.

2. ui.js разделен на отдельные UI-модули.
   Теперь:
   - public/js/ui-utils.js — общие UI-хелперы, панели, текст, кнопки, прогресс-бары.
   - public/js/ui-jobs.js — HUD, панели работ, мусорки и подтверждения действий.
   - public/js/ui-dev.js — dev mode, чекбоксы, координаты.
   - public/js/ui-avatar-shop.js — редактор персонажа и магазин.
   - public/js/ui-profile.js — профиль игрока.
   - public/js/ui-chat.js — чат, ЛС, контакты, контекстное меню.
   - public/js/ui-inventory.js — инвентарь и уведомления.
   - public/js/ui-auth.js — авторизация/регистрация и DOM-поля.
   - public/js/ui.js — оставлен как legacy-placeholder.

3. index.html обновлен.
   Скрипты подключаются в явном порядке:
   core/player/network → city/map/jobs → render → UI modules → render-loop.

4. package.json обновлен.
   npm run check теперь проверяет новые модули.

Проверка:
- npm run check: успешно.
- Проверка всех подключенных script src из index.html: пропусков нет.
- server.js и local-server.js не менялись по логике.

Важно:
Это безопасный рефактор структуры. Он не переводит проект на ES modules/import-export, чтобы не сломать текущий браузерный запуск и Socket.IO. Все функции остались в совместимой глобальной среде, но файлы теперь логически разделены.
